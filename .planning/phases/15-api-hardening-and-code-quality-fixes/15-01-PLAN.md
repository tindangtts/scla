<objective>
Secure the three upgrade-request endpoints in auth.ts that are currently unauthenticated, and wrap the two-step approve/reject operations in database transactions to eliminate partial-update failures.

Purpose: Unauthenticated admin endpoints are a critical security hole — any client can list, approve, or reject resident upgrade requests. The multi-step approve/reject (update upgradeRequestsTable THEN update usersTable) has no transaction wrapping, meaning a crash between the two writes leaves data inconsistent.
Output: auth.ts with admin-gated upgrade endpoints and transactional approve/reject handlers.
</objective>

**Change 2 — POST /upgrade-requests/:id/approve (currently line ~175):**
a) Add admin auth guard as the first two lines of the handler body.
b) Wrap the two `db.update` calls in a `db.transaction`. The transaction receives a `tx` parameter — replace `db.update(...)` with `tx.update(...)` inside the transaction. Also move the `db.select` for fetching the request BEFORE the transaction (it's a read, no need to be in tx). Structure:
```typescript
router.post("/upgrade-requests/:id/approve", async (req, res) => {
  const adminPayload = requireAdmin(req, res);
  if (!adminPayload) return;

  const { id } = req.params;
  const [request] = await db.select()
    .from(upgradeRequestsTable)
    .where(eq(upgradeRequestsTable.id, id))
    .limit(1);
  if (!request) {
    return res.status(404).json({ error: "not_found", message: "Request not found" });
  }

  const updated = await db.transaction(async (tx) => {
    const [updatedRequest] = await tx.update(upgradeRequestsTable)
      .set({ status: "approved", reviewedAt: new Date() })
      .where(eq(upgradeRequestsTable.id, id))
      .returning();

    await tx.update(usersTable)
      .set({
        userType: "resident",
        upgradeStatus: "approved",
        unitNumber: request.unitNumber,
        residentId: request.residentId,
        developmentName: request.developmentName,
      })
      .where(eq(usersTable.id, request.userId));

    return updatedRequest;
  });

  return res.json({
    id: updated.id,
    userId: updated.userId,
    userName: updated.userName,
    userEmail: updated.userEmail,
    unitNumber: updated.unitNumber,
    residentId: updated.residentId,
    developmentName: updated.developmentName,
    status: updated.status,
    submittedAt: updated.submittedAt.toISOString(),
    reviewedAt: updated.reviewedAt?.toISOString() ?? null,
    reviewNote: updated.reviewNote ?? null,
  });
});
```
