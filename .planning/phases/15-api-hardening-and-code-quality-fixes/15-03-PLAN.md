<objective>
Replace the count-then-increment race condition for booking and ticket number generation with atomic PostgreSQL sequences, and update the Drizzle schema to declare those sequences.

Purpose: The current pattern (`count = SELECT * FROM table WHERE userId=X; number = count.length + 1`) has a race condition — two concurrent requests read the same count and generate duplicate numbers. PostgreSQL `nextval()` is atomic and guarantees uniqueness.
Output: Two new pg sequences declared in Drizzle schema; booking/ticket creation routes use `nextval` instead of count-based generation.
</objective>
