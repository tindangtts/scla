<task type="auto">
  <name>Task 2: Create GitHub Actions CI/CD workflow</name>
  <files>.github/workflows/ci.yml</files>
  <read_first>package.json, .replit, artifacts/api-server/vitest.config.ts</read_first>
  <action>
Create `.github/workflows/ci.yml` with the following structure:

**Name:** `CI/CD`

**Triggers:**
- `push` to `main` branch
- `pull_request` to `main` branch

**Jobs:**

**Job 1: `validate`** (runs on all pushes and PRs)
- runs-on: `ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. Install pnpm: `pnpm/action-setup@v4` with `version: 9` (read from packageManager field or use 9)
  3. `actions/setup-node@v4` with `node-version: 24`, cache: `pnpm`
  4. `pnpm install --frozen-lockfile`
  5. Step "Lint (Prettier)" — `pnpm run format:check`
  6. Step "Type Check" — `pnpm run typecheck`
  7. Step "Tests" — `pnpm run test:ci` with env `DATABASE_URL: postgresql://test:test@localhost:5432/test_db` and `NODE_ENV: test`
  - Add a `services` block for PostgreSQL 16:
    ```yaml
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
          POSTGRES_DB: test_db
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
    ```
  - Before running tests, add a step to push the DB schema: `pnpm --filter db push` with the same DATABASE_URL env var

**Job 2: `deploy`** (runs only on push to main, after validate succeeds)
- runs-on: `ubuntu-latest`
- needs: `validate`
- if: `github.event_name == 'push' && github.ref == 'refs/heads/main'`
- Steps:
  1. `actions/checkout@v4`
  2. Step "Trigger Replit Deploy" — Use `curl` to call the Replit deploy hook. The URL should come from a GitHub secret `REPLIT_DEPLOY_HOOK_URL`. Use: `curl -X POST "$REPLIT_DEPLOY_HOOK_URL"` with env from `secrets.REPLIT_DEPLOY_HOOK_URL`. If the secret is not set, the step should still succeed (add `continue-on-error: true` and a note that the secret must be configured).

Add a comment at the top of the file explaining:
- CICD-01: validate job runs lint, typecheck, tests on every push/PR
- CICD-02: GitHub branch protection rules must be configured to require the validate job to pass before merging (manual step — documented in comment)
- CICD-03: deploy job triggers Replit redeploy on merge to main

For CICD-02 (merge blocking), add a comment block in the workflow file:
```yaml
# IMPORTANT: To enforce merge blocking (CICD-02), configure GitHub branch protection:
# Settings > Branches > Branch protection rules > Add rule for "main":
#   - Require status checks to pass: check "validate"
#   - Require branches to be up to date before merging
```
