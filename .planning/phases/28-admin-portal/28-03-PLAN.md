**1. Create `artifacts/web/src/lib/queries/admin-staff.ts`:**
- `getAllStaff()`: SELECT all from staffUsersTable order by createdAt desc. Return StaffUser[].
- `getStaffById(id: string)`: SELECT by id. Return StaffUser | null.

**2. Create `artifacts/web/src/lib/queries/admin-audit.ts`:**
- `getAuditLogs(actionFilter?: string, dateFrom?: string, dateTo?: string)`: SELECT all from auditLogsTable with optional filters. If actionFilter provided, eq on action column. If dateFrom/dateTo provided, use gte/lte on createdAt (parse ISO strings to Date). Order by createdAt desc. Limit 100. Return AuditLog[].

**3. Create `artifacts/web/src/app/(admin)/staff/actions.ts`:**
- `"use server"`, import requireAdmin, db, schema, bcryptjs, revalidatePath, redirect
- `createStaff(formData: FormData)`:
  - Extract name, email, password, role from formData
  - Hash password with bcryptjs (10 rounds) — import `bcryptjs` (already a project dependency from Phase 25)
  - Insert into staffUsersTable
  - Also create Supabase Auth user via admin client (same pattern as upgrade-requests/actions.ts) with user_metadata: { user_type: "staff", staff_role: role }
  - Insert audit log: action="staff_create", targetType="staff", targetId=new staff id
  - Redirect to /admin/staff
- `updateStaff(formData: FormData)`:
  - Extract staffId, name, role, isActive (checkbox)
  - Update staffUsersTable SET name, role, isActive, updatedAt
  - Insert audit log: action="staff_update" (or "staff_deactivate" if isActive changed to false)
  - revalidatePath("/admin/staff")
- `deleteStaff` is NOT needed — use deactivate via isActive=false instead

**1. Create `artifacts/web/src/lib/queries/admin-wallets.ts`:**
- `searchResidents(search?: string)`: SELECT from usersTable WHERE userType='resident'. If search provided, ILIKE on name or email. Order by name. Return User[]. Used for resident picker.
- `getResidentWalletDetail(userId: string)`: Returns { user, balance, recentTransactions }.
  - user: SELECT from usersTable WHERE id
  - balance: SUM wallet transactions (credit - debit) for user, cast to text, coalesce to '0'
  - recentTransactions: SELECT last 20 from walletTransactionsTable WHERE userId, order by createdAt desc

**2. Create `artifacts/web/src/app/(admin)/wallets/page.tsx`:**
- `export const dynamic = "force-dynamic"`, `requireAdmin()`
- Accept searchParams for `search`
- Import `searchResidents`
- Render search form (GET): Input name="search" placeholder="Search residents by name or email"
- If search provided, render results as a list of Cards:
  - Each card: Resident name, email, unit number, development
  - "Manage Wallet" link to /admin/wallets/{userId}
- If no search, show instruction text: "Search for a resident to manage their wallet"

**3. Create `artifacts/web/src/app/(admin)/wallets/[userId]/page.tsx`:**
- `export const dynamic = "force-dynamic"`, `requireAdmin()`
- Extract userId from params
- Import `getResidentWalletDetail`
- If user not found, `notFound()`
- Render:
  - Header: Resident name + email
  - Balance card: Current balance (large text, formatted)
  - Adjustment form card: 
    - Type: select (credit/debit)
    - Amount: Input type="number" step="0.01" min="0.01"
    - Description: Input (required — e.g. "Monthly allowance", "Penalty charge")
    - Reason: Input (optional — internal note)
    - Submit button bound to `adjustWallet` action
    - Hidden inputs: userId
  - Recent transactions table: Date, Type (Badge — credit=green, debit=red), Amount, Description, Reason
- Back link to /admin/wallets

**4. Create `artifacts/web/src/app/(admin)/wallets/actions.ts`:**
- `"use server"`, import requireAdmin, db, schema, revalidatePath
- `adjustWallet(formData: FormData)`:
  - Extract userId, type (credit|debit), amount, description, reason from formData
  - Validate: amount must be positive number, type must be credit or debit, description required
  - Call `requireAdmin()` to get staff info
  - Insert into walletTransactionsTable: { userId, type, amount, description, reason, category: "admin_adjustment", reference: `ADJ-${Date.now()}` }
  - Insert into auditLogsTable: { actorId: staff.id, actorEmail: staff.email, action: "wallet_adjust", targetType: "user", targetId: userId, metadata: { type, amount, description, reason } }
  - revalidatePath(`/admin/wallets/${userId}`)
