### Root cause of TS2556

`vi.fn()` returns a `Mock<Procedure>` whose call signature expands to positional params (e.g. `(args_0: unknown, args_1: unknown) => ...`), not a rest-param signature. Spreading `unknown[]` into a positional-param signature violates TS2556.

### Canonical fix for TS2556 (use Option A — minimal diff)

Option A — drop the spread indirection (preferred):
```ts
vi.mock("@/lib/db", () => ({
  db: {
    select: mockSelect,        // was: (...args: unknown[]) => mockSelect(...args)
    insert: mockInsert,
    delete: mockDelete,
  },
}));
```

Option B — keep factory, narrow args (fallback if a file needs factory logic):
```ts
select: (...args: Parameters<typeof mockSelect>) => mockSelect(...args),
```

None of the six files perform logic beyond delegation, so Option A applies everywhere.

### Canonical fix for TS2345 (ticket messages, line 63)

The error is:
```ts
function makeRequest(url: string, options?: RequestInit) {
  return new NextRequest(new URL(url, "http://localhost:3000"), options);
  //                                                            ^^^^^^^ TS2345
}
```
Root cause: DOM lib `RequestInit` types `signal` as `AbortSignal | null`; Next's `RequestInit` types it as `AbortSignal | undefined`.

Primary fix (typed via Next's RequestInit):
```ts
import type { RequestInit as NextRequestInit } from "next/dist/server/web/spec-extension/request";

function makeRequest(url: string, options?: NextRequestInit) {
  return new NextRequest(new URL(url, "http://localhost:3000"), options);
}
```

Fallback fix (if the deep import path does not resolve):
```ts
function makeRequest(url: string, options?: RequestInit) {
  return new NextRequest(
    new URL(url, "http://localhost:3000"),
    options as ConstructorParameters<typeof NextRequest>[1],
  );
}
```
Prefer the fallback if the deep import fails resolution. Do NOT use `options as any`.
