<objective>
Add per-IP rate limiting to authentication endpoints to block brute-force login attacks.

Purpose: Without rate limiting, an attacker can test millions of password combinations against `/api/auth/login`. This plan caps login attempts at 5 per minute per IP and returns a 429 with a clear error message and `Retry-After` header.

Output: A reusable `authRateLimiter` middleware applied to the three auth endpoints. Other API endpoints are unaffected.
</objective>

```typescript
const limiter = rateLimit({
  windowMs: 60 * 1000,       // 1 minute window
  max: 5,                     // max requests per windowMs per IP
  standardHeaders: "draft-7", // Adds RateLimit-* headers (RFC 6585)
  legacyHeaders: false,
  handler: (req, res) => {
    res.status(429).json({

      message: "Too many login attempts. Please try again in 1 minute.",

    });
  },
});
```

```typescript
import rateLimit from "express-rate-limit";

/**
 * Rate limiter for authentication endpoints.
 * Allows 5 requests per IP per minute (per D-05).
 * Returns 429 with a meaningful error and Retry-After header on breach.
 * In-memory store — acceptable for single-instance Replit deployment (per D-06).
 */
export const authRateLimiter = rateLimit({
  windowMs: 60 * 1000, // 1-minute rolling window
  max: 5,              // 5 attempts per window per IP
  standardHeaders: "draft-7", // Adds RateLimit-Limit, RateLimit-Remaining, RateLimit-Reset headers
  legacyHeaders: false,       // Disable X-RateLimit-* headers
  handler: (_req, res) => {
    res.status(429).json({
      error: "too_many_requests",
      message: "Too many attempts. Please wait 1 minute before trying again.",
    });
  },
});
```
