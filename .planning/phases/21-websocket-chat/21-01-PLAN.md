    1. Install ws: `cd artifacts/api-server && pnpm add ws && pnpm add -D @types/ws`

    2. Create `artifacts/api-server/src/lib/ws-server.ts` with these exports:
       - `setupWebSocket(server: http.Server): void` — creates `new WebSocketServer({ server })`, handles upgrade
       - `broadcastToTicket(ticketId: string, message: TicketMessage): void` — sends to all clients in a ticket room

       Implementation details:
       - On `connection`, parse URL query params: `?token=xxx&ticketId=yyy&role=resident|staff`
       - For `role=resident`: verify token with `jwt.verify(token)` from `../lib/jwt.js`. Extract userId.
       - For `role=staff`: verify token with the same HMAC SHA256 logic used in admin.ts (ctx: "admin" header check). Extract staffId.
       - If token invalid, send JSON `{ type: "error", message: "unauthorized" }` and close with code 1008.
       - If valid, add the socket to a Map<string, Set<WebSocket>> keyed by ticketId (the "room").
       - On `close`, remove socket from room. If room empty, delete the key.
       - On incoming `message` from client: ignore (clients send messages via REST POST, WS is receive-only for simplicity).
       - `broadcastToTicket` iterates the room set and sends JSON: `{ type: "new_message", payload: TicketMessage }` to each open socket.
       - Add a 30-second heartbeat ping interval. On pong timeout, terminate stale connections.

    3. Modify `artifacts/api-server/src/index.ts`:
       - Import `setupWebSocket` from `./lib/ws-server.js`
       - `app.listen()` returns `http.Server`. Capture it: `const server = app.listen(port, ...)`
       - After the `logger.info` line, call `setupWebSocket(server)`

       The current code uses a callback form of `app.listen`. Restructure to:
       ```typescript
       const server = app.listen(port, () => {
         logger.info({ port }, "Server listening");
         setupWebSocket(server);
         startScheduler();
       });
       server.on("error", (err) => {
         logger.error({ err }, "Error listening on port");
         process.exit(1);
       });
       ```
       Note: Express 5 app.listen may return http.Server directly. The callback on listen is for the 'listening' event. Use the pattern above to ensure server is available for WS attachment.

       ```
       // GET /admin/tickets/:id/messages — staff reads chat messages for any ticket
       router.get("/tickets/:id/messages", async (req, res) => {
         const payload = requireAdmin(req, res);
         if (!payload) return;
         const messages = await db.select().from(ticketMessagesTable)
           .where(eq(ticketMessagesTable.ticketId, req.params.id))
           .orderBy(asc(ticketMessagesTable.createdAt));
         return res.json(messages);
       });

       // POST /admin/tickets/:id/messages — staff sends a chat message
       router.post("/tickets/:id/messages", async (req, res) => {
         const payload = requireAdmin(req, res);
         if (!payload) return;
         const { content } = req.body;
         if (!content || !content.trim()) {
           return res.status(400).json({ error: "validation_error", message: "content is required" });
         }
         const [inserted] = await db.insert(ticketMessagesTable).values({
           ticketId: req.params.id,
           senderId: payload.staffId,
           senderType: "staff",
           content: content.trim(),
         }).returning();
         broadcastToTicket(req.params.id, inserted);
         return res.status(201).json(inserted);
       });
       ```
