Create query helpers following the established pattern in `src/lib/queries/`:

**facilities.ts:**
- `getFacilities(category?: string)`: Select all from facilitiesTable where isAvailable=true, optionally filter by category. Order by name.
- `getFacilityById(id: string)`: Select one facility by ID, return first row or null.
- `getAvailableSlots(facilityId: string, date: string)`: Generate hourly slots from facility openingTime to closingTime (e.g. 06:00-22:00 produces slots 06:00-07:00, 07:00-08:00, ..., 21:00-22:00). Then query bookingsTable for that facility+date where status != 'cancelled' to find booked slots. Return array of `{ startTime: string, endTime: string, isBooked: boolean }`.

**bookings.ts:**
- `getNextBookingNumber()`: Query count from bookingsTable, format as `BK-${String(count+1).padStart(4, '0')}`.
- `getUserBookings(userId: string, status?: string)`: Select from bookingsTable where userId matches, optionally filter by status. Order by date desc, startTime desc.
- `getBookingById(id: string, userId: string)`: Select one booking scoped to user.

**Facilities list page** at `/bookings/facilities/page.tsx`:
- Server Component with `force-dynamic`.
- Accept `searchParams` with optional `category` filter.
- Show category filter badges (All + each facility_category enum value, display labels like "Swimming Pool", "Tennis Court", etc.).
- Render each facility as a Card with name, description, category badge, member rate in MMK, and a Link to `/bookings/facilities/${id}`.

**Facility detail page** at `/bookings/facilities/[id]/page.tsx`:
- Server Component showing facility info (name, description, rates, hours, capacity).
- Accept `searchParams.date` (default to today's date in YYYY-MM-DD).
- Call `getAvailableSlots(id, date)` and pass to BookingForm client component.
- Show date navigation (previous day / next day links, but not before today).

**BookingForm** client component at `/bookings/facilities/[id]/booking-form.tsx`:
- Display available slots as a grid of buttons. Booked slots are disabled/grayed out. Available slots are selectable.
- User picks one slot (highlight selected).
- Toggle for "Repeat weekly" with a select for number of weeks (4, 8, 12 options).
- Optional notes textarea.
- Submit button triggers `bookSlot` Server Action via `useActionState`.
- Display success message with booking number or error message.

**bookSlot Server Action** at `/bookings/facilities/[id]/book-action.ts`:
- Validate: user authenticated via requireAuth(), look up DB user, facility exists and isAvailable.
- Validate selected slot is not already booked (re-check in action to prevent race conditions).
- Generate booking number via `getNextBookingNumber()`.
- If recurring: generate a `recurringGroupId` (use `crypto.randomUUID()`), create N bookings (1 per week) for the same day-of-week and time slot, each with incrementing booking numbers. Skip any weeks where the slot is already booked (log a warning but continue).
- If single: create one booking record.
- Set totalAmount = facility.memberRate per slot (use memberRate for all residents for now -- per Claude's discretion).
- Revalidate path `/bookings` and redirect to `/bookings` on success.
