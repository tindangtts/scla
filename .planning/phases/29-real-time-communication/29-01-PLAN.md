  <action>
1. Add `ws` dependency to artifacts/web/package.json: `pnpm add ws` and `pnpm add -D @types/ws` in the web workspace.

2. Create `artifacts/web/src/lib/ws-server.ts` — a standalone WebSocket server:
   - Imports `ws` (WebSocketServer)
   - Listens on port 3002 (or WS_PORT env var)
   - On connection, client sends JSON: `{ type: "join", ticketId: string }`
   - Server maintains a Map<ticketId, Set<WebSocket>> for room management
   - When a message is broadcast (from REST API or direct), server sends to all clients in that ticketId room: `{ type: "message", data: TicketMessage }`
   - On disconnect, removes client from room
   - Export `startWsServer()` function and a `broadcastToTicket(ticketId: string, message: TicketMessage)` function
   - Add a `ws-server.start.ts` entry script that calls startWsServer() (runnable via `npx tsx artifacts/web/src/lib/ws-server.start.ts`)

3. Create `artifacts/web/src/app/api/tickets/[id]/messages/route.ts`:
   - GET handler: Requires auth via createClient() (check Supabase session from cookies). Fetches messages from ticketMessagesTable where ticketId matches, ordered by createdAt asc. Returns JSON array.
   - POST handler: Requires auth. Accepts `{ content: string }`. Determines senderId from the authenticated user's app DB row (look up usersTable by email). Determines senderType: check if user is in staffUsersTable — if yes, "staff"; else "resident". Inserts into ticketMessagesTable. Returns the created message as JSON. Does NOT call broadcastToTicket here (WS server is a separate process).
   - Validate ticketId is a valid UUID. Return 400 for empty content, 404 for invalid ticket.

4. Create `artifacts/web/src/hooks/use-ticket-chat.ts` — a React hook:
   - `useTicketChat(ticketId: string, initialMessages: TicketMessage[])` returns `{ messages, sendMessage, connectionStatus }` where connectionStatus is "connected" | "polling" | "disconnected"
   - On mount, tries to connect to `ws://localhost:3002` (or NEXT_PUBLIC_WS_URL env var). Sends `{ type: "join", ticketId }`.
   - On WS message of type "message", appends to messages state (dedup by id).
   - On WS error/close, sets connectionStatus to "polling" and starts a setInterval polling GET /api/tickets/{ticketId}/messages every 5 seconds. Compares with current messages by checking if response contains messages with IDs not in current state.
   - `sendMessage(content: string)` does POST /api/tickets/{ticketId}/messages. On success, appends returned message to local state immediately.
   - On unmount, closes WS connection and clears polling interval.
   - When WS reconnects successfully, stops polling and sets status back to "connected".
   - Reconnection: on WS close, attempt reconnect after 3 seconds, up to 5 retries, then stay in polling mode.
