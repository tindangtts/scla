  <action>
1. Install dependencies in web workspace: `pnpm add web-push resend` and `pnpm add -D @types/web-push` in artifacts/web.

2. Create `artifacts/web/public/sw.js` — plain JS service worker (not TypeScript):
   - Listen for "push" event: extract data with `event.data.json()` which contains `{ title, body, url }`. Show notification with `self.registration.showNotification(title, { body, icon: "/icon-192.png", data: { url } })`.
   - Listen for "notificationclick" event: close notification, open the url from notification.data via `clients.openWindow(notification.data.url)`.

3. Create `artifacts/web/src/app/api/push/subscribe/route.ts`:
   - POST handler: Requires auth (create Supabase client, getUser). Accepts JSON body: `{ endpoint, p256dh, auth }`. Look up user in usersTable by email to get userId. Upsert into pushSubscriptionsTable (on conflict endpoint, update p256dh and auth). Return 200.
   - DELETE handler: Requires auth. Accepts JSON body `{ endpoint }`. Delete from pushSubscriptionsTable where endpoint matches and userId matches. Return 200.

4. Create `artifacts/web/src/lib/push.ts` — server-side push sender:
   - Import web-push. Configure VAPID: `webpush.setVapidDetails("mailto:noreply@starcityliving.com", process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!, process.env.VAPID_PRIVATE_KEY!)`.
   - Export `async function sendPushToUser(userId: string, payload: { title: string; body: string; url: string })`: Query pushSubscriptionsTable for all subscriptions with that userId. For each, call `webpush.sendNotification(subscription, JSON.stringify(payload))`. On 410 (gone) status, delete the subscription from DB. Wrap in try/catch per subscription — don't fail all if one fails.

5. Create `artifacts/web/src/components/push-prompt.tsx` — "use client" component:
   - On mount, check if `"Notification" in window && Notification.permission === "default"`. If so, after 3 second delay, show a small banner: "Enable push notifications to stay updated on your tickets and bills" with "Enable" and "Dismiss" buttons.
   - On "Enable": call `Notification.requestPermission()`. If granted, register service worker (`navigator.serviceWorker.register("/sw.js")`), get push subscription via `registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY) })`. POST the subscription keys (endpoint, p256dh, auth) to `/api/push/subscribe`. Hide banner.
   - On "Dismiss": hide banner, set localStorage flag `push-prompt-dismissed` to prevent showing again.
   - If permission already "granted", silently register SW on mount (no banner).
   - Include `urlBase64ToUint8Array` helper function inline.

6. Update `artifacts/web/src/app/(resident)/layout.tsx`:
   - Import PushPrompt from @/components/push-prompt
   - Add `<PushPrompt />` inside the layout, after the header element.
  </action>

   - Import Resend from "resend". Initialize: `const resend = new Resend(process.env.RESEND_API_KEY)`.
   - Export `async function sendBillOverdueEmail(to: string, userName: string, invoiceNumber: string, amount: number)`:
     Sends email with `resend.emails.send({ from: "Star City Living <noreply@starcityliving.com>", to, subject: "Bill Overdue: ${invoiceNumber}", html: ... })`. HTML body: simple styled email with heading "Bill Payment Reminder", body "Dear {userName}, your invoice {invoiceNumber} for MMK {amount} is overdue. Please log in to make payment.", link to /bills.
   - Export `async function sendTicketUpdateEmail(to: string, userName: string, ticketNumber: string, newStatus: string)`:
     Similar structure. Subject: "Ticket Update: {ticketNumber}". Body: "Your ticket {ticketNumber} status has been updated to {newStatus}."
   - Both functions check if RESEND_API_KEY is set; if not, log a warning and return silently (graceful degradation).

2. Create `artifacts/web/src/lib/notifications.ts` — unified notification trigger:
   - This is the server-side "fire notification" module that other Server Actions will call.
   - Import: sendPushToUser from ./push, sendBillOverdueEmail/sendTicketUpdateEmail from ./email, db, notificationsTable, usersTable.
   - Export `async function notifyBillOverdue(userId: string, invoiceNumber: string, amount: number)`:
     a. Insert into notificationsTable: title "Bill Overdue", body "Invoice {invoiceNumber} for MMK {amount} is overdue", type "ticket_update" (reuse closest enum — or use "general"), relatedId null.
     b. Call sendPushToUser(userId, { title: "Bill Overdue", body: "Invoice {invoiceNumber} is overdue", url: "/bills" }).
     c. Look up user email from usersTable. If user.emailNotifications is true, call sendBillOverdueEmail(email, name, invoiceNumber, amount).
     d. All three fire-and-forget with individual try/catch — don't block the calling action.
   - Export `async function notifyTicketUpdate(userId: string, ticketId: string, ticketNumber: string, newStatus: string)`:
     a. Insert notification: title "Ticket Updated", body "Ticket {ticketNumber} is now {newStatus}", type "ticket_update", relatedId ticketId.
     b. sendPushToUser(userId, { title: "Ticket Updated", body: ..., url: `/star-assist/${ticketId}` }).
     c. If emailNotifications, sendTicketUpdateEmail.
   - Export `async function notifyNewMessage(userId: string, ticketId: string, ticketNumber: string)`:
     a. Insert notification: title "New Message", body "New message on ticket {ticketNumber}", type "ticket_update", relatedId ticketId.
     b. sendPushToUser.
     c. No email for chat messages (too noisy).
