Create a new API route at `artifacts/web/src/app/api/cron/bill-overdue-check/route.ts`.

This route:

1. **Authentication**: Accepts a `CRON_SECRET` header for security. Compare against `process.env.CRON_SECRET`. If CRON_SECRET env var is not set, allow the request (development mode). If set, reject requests without a matching header with 401.

2. **Query**: Find all invoices where `status = 'unpaid'` AND `dueDate < current_date` (same logic as the existing overdue filter in `lib/queries/bills.ts`).

3. **Deduplicate per user**: Group overdue invoices by userId. For each user, call `notifyBillOverdue(userId, invoiceNumber, totalAmount)` for each overdue invoice. To avoid spamming, only process invoices whose dueDate is exactly 1 day past (i.e., `dueDate = current_date - interval '1 day'`) OR allow a `?force=true` query param to process ALL overdue invoices.

4. **Response**: Return JSON `{ processed: number, errors: number }` with 200 status.

Concrete implementation:

```typescript
import { NextRequest, NextResponse } from "next/server";
import { db } from "@/lib/db";
import { invoicesTable } from "@workspace/db/schema";
import { eq, and, lt, sql } from "drizzle-orm";
import { notifyBillOverdue } from "@/lib/notifications";

export async function POST(request: NextRequest) {
  // Auth check
  const cronSecret = process.env.CRON_SECRET;
  if (cronSecret && request.headers.get("x-cron-secret") !== cronSecret) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  const force = request.nextUrl.searchParams.get("force") === "true";

  // Find overdue invoices
  const conditions = [
    eq(invoicesTable.status, "unpaid"),
  ];

  if (force) {
    // All overdue: dueDate < today
    conditions.push(lt(invoicesTable.dueDate, sql`current_date`));
  } else {
    // Only newly overdue: dueDate was yesterday
    conditions.push(
      sql`${invoicesTable.dueDate} = current_date - interval '1 day'`
    );
  }

  const overdueInvoices = await db
    .select({
      id: invoicesTable.id,
      userId: invoicesTable.userId,
      invoiceNumber: invoicesTable.invoiceNumber,
      totalAmount: invoicesTable.totalAmount,
    })
    .from(invoicesTable)
    .where(and(...conditions));

  let processed = 0;
  let errors = 0;

  for (const invoice of overdueInvoices) {
    try {
      await notifyBillOverdue(
        invoice.userId,
        invoice.invoiceNumber,
        Number(invoice.totalAmount)
      );
      processed++;
    } catch {
      errors++;
    }
  }

  return NextResponse.json({ processed, errors, total: overdueInvoices.length });
}
```

Key constraints:
- Import `notifyBillOverdue` from `@/lib/notifications` (the existing fully-implemented function)
- Use the same overdue logic as `lib/queries/bills.ts`: status=unpaid AND dueDate < current_date
- Default behavior: only notify for invoices that became overdue yesterday (prevents re-notifying)
- `?force=true` param: notify for ALL overdue invoices (useful for initial catch-up or testing)
- Each notification is independent; one failure doesn't block others
