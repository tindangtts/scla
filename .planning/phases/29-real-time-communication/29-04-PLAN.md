Modify ws-server.ts to add an internal HTTP server alongside the WebSocket server. This HTTP server listens on WS_PORT + 1 (default 3003) and exposes a single endpoint:

POST /broadcast
- Body: JSON `{ ticketId: string, message: TicketMessage }`
- Calls the existing `broadcastToTicket(ticketId, message)` function
- Returns 200 on success
- Returns 400 if ticketId or message is missing

Implementation details:
- Use Node.js built-in `http.createServer` (no extra dependency needed) to create a minimal HTTP server
- Parse JSON body manually with `req.on('data')` and `req.on('end')` (standard Node.js pattern)
- Start the HTTP server inside `startWsServer()` so both run together
- The HTTP endpoint is internal-only (localhost), not exposed to the internet
- Add env var `WS_BROADCAST_PORT` defaulting to 3003

The function signature of `startWsServer` stays the same. The HTTP server is a side-effect started within it.

In the POST handler of `/api/tickets/[id]/messages/route.ts`, after the message is inserted into the DB (line 130-138) and before the notification block, add a fire-and-forget HTTP call to the WS server's broadcast endpoint:

```typescript
// Broadcast to WebSocket clients (fire-and-forget, cross-process)
try {
  const wsBroadcastUrl = process.env.WS_BROADCAST_URL || "http://localhost:3003/broadcast";
  fetch(wsBroadcastUrl, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ ticketId, message }),
  }).catch(() => {});
} catch {
  // Don't block response if WS server is unavailable
}
```

Place this AFTER `const [message] = await db.insert(...)` and BEFORE the notification try/catch block (around line 141).

Key constraints:
- Use native `fetch` (available in Next.js server environment)
- Fire-and-forget: do NOT await the fetch, wrap in .catch(() => {}) so failures are silent
- Use env var `WS_BROADCAST_URL` so the URL is configurable (default http://localhost:3003/broadcast)
- This is a cross-process call, not an import — the WS server runs in a separate Node.js process
