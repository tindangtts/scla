    **Create ticket query helpers** `src/lib/queries/tickets.ts`:
    - `getTickets(userId: string, status?: string)` — query ticketsTable WHERE userId, optionally filtered by status. Order by createdAt DESC. Return array of Ticket.
    - `getTicketById(id: string, userId: string)` — query single ticket WHERE id AND userId. Return Ticket | null.
    - `getTicketMessages(ticketId: string)` — query ticketMessagesTable WHERE ticketId ORDER BY createdAt ASC. Return array of TicketMessage.
    - `getNextTicketNumber()` — query ticketsTable COUNT, return formatted string `"SA-" + String(count + 1).padStart(4, "0")`.

    Use `import { db } from "@/lib/db"` and drizzle operators.

    **Create ticket list page** `(resident)/star-assist/page.tsx`:
    - `export const dynamic = "force-dynamic"`.
    - Accept `searchParams` prop with optional `status` field (type: `Promise<{ status?: string }>`).
    - Call `requireAuth()`, look up dbUser by email from usersTable.
    - Call `getTickets(dbUser.id, status)`.
    - Render filter tabs at top: All, Open, In Progress, Completed, Closed — each links to `/star-assist?status=X`. Highlight active with Badge variant.
    - Render "New Ticket" button/link at top right linking to `/star-assist/new`.
    - Render tickets as Card list. Each card shows: ticketNumber (bold), title, category badge, status badge (open=default, in_progress=secondary, completed=outline, closed=destructive), date. Card links to `/star-assist/{id}`.
    - Empty state: "No tickets found. Create your first ticket!" with link to /star-assist/new.

    **Create ticket detail page** `(resident)/star-assist/[id]/page.tsx`:
    - `export const dynamic = "force-dynamic"`.
    - Accept `params` prop (`Promise<{ id: string }>`).
    - Call `requireAuth()`, look up dbUser, call `getTicketById(id, dbUser.id)`.
    - If not found, call `notFound()`.
    - Render ticket header: ticketNumber, status Badge, category Badge, createdAt date.
    - Render description in a Card.
    - If attachmentUrl exists, render the image (base64 data URL) in an img tag with max-width.
    - Render updates/messages section: list the `updates` JSON array entries chronologically, showing author, message, date. Also query and display ticketMessagesTable entries below (for Phase 29 real-time chat readiness).
    - Note at bottom: "Real-time chat coming soon" placeholder text (Phase 29 will add WebSocket).

    - Call `getNextTicketNumber()` from queries/tickets.ts.
