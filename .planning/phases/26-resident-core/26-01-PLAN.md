    **Create pay invoice Server Action** `bills/[id]/pay-action.ts`:
    - `"use server"` directive.
    - Export `payInvoice(prevState: {error?: string; success?: boolean}, formData: FormData)` for useActionState.
    - Extract `invoiceId` from formData.
    - Call `requireAuth()`, look up dbUser by email.
    - Fetch the invoice by id, verify it belongs to dbUser.id and status !== 'paid'. Return error if not found or already paid.
    - Compute wallet balance using `getWalletBalance(dbUser.id)` from queries/wallet.ts.
    - Compute remaining amount: `Number(invoice.totalAmount) - Number(invoice.paidAmount)`.
    - If wallet balance < remaining amount, return `{ error: "Insufficient wallet balance" }`.
    - Insert a debit wallet_transaction: `{ userId: dbUser.id, type: "debit", amount: remainingAmount.toFixed(2), description: "Payment for invoice " + invoice.invoiceNumber, reference: invoice.id, category: "payment" }`.
    - Update invoice: set paidAmount = totalAmount, status = "paid".
    - Call `revalidatePath("/bills")` and `revalidatePath("/bills/" + invoiceId)` and `revalidatePath("/wallet")` and `revalidatePath("/")`.
    - Return `{ success: true }`.

    **Create PayButton client component** `bills/[id]/pay-button.tsx`:
    - `"use client"` directive.
    - Props: `{ invoiceId: string, amount: string }` (the remaining amount to pay, formatted).
    - Uses `useActionState` from react with `payInvoice` action.
    - Renders a form with hidden input name="invoiceId" value={invoiceId}.
    - Button text: "Pay {amount} MMK". Shows "Processing..." when pending (useActionState pending state).
    - Display error message from state.error in red text above button.
    - Display success message "Payment successful!" in green when state.success is true.

    **Create profile page** `(resident)/profile/page.tsx`:
    - `export const dynamic = "force-dynamic"`.
    - Call `requireAuth()`, look up dbUser by email.
    - If dbUser not found, show error card.
    - Render profile card showing: name, email (read-only), phone, user type badge, unit number, resident ID, development name, member since date.
    - Import and render `EditForm` client component below the info card, passing current dbUser values as props.

    **Create profile update Server Action** `profile/actions.ts`:
    - `"use server"` directive.
    - Export `updateProfile(prevState: {error?: string; success?: boolean}, formData: FormData)`.
    - Extract name, phone from formData. Validate: name must be 2-100 chars, phone must be non-empty.
    - Call `requireAuth()`, look up dbUser by email.
    - Update usersTable SET name, phone, updatedAt = new Date() WHERE id = dbUser.id.
    - Call `revalidatePath("/profile")` and `revalidatePath("/")`.
    - Return `{ success: true }`.

    **Create EditForm client component** `profile/edit-form.tsx`:
    - `"use client"` directive.
    - Props: `{ name: string, phone: string }`.

    - Submit button: "Save Changes", shows "Saving..." when pending.
    - Display error/success messages.
    - Use Label + Input from shadcn/ui components, consistent with upgrade-form.tsx pattern.

    Follow the established pattern from upgrade-form.tsx: separate client component with useActionState, Server Action with prevState signature, form with hidden fields where needed.
