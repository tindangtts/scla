    - CI does NOT run E2E tests (requires running server + DB — separate concern)

  </acceptance_criteria>
  <action>
    1. Create .github/workflows/ directory if it doesn't exist.

    2. Create .github/workflows/ci.yml:
       ```yaml
       name: CI
       on:
         push:
           branches: [main]
         pull_request:
           branches: [main]
       
       jobs:
         lint-typecheck-test:
           runs-on: ubuntu-latest
           steps:
             - uses: actions/checkout@v4
             - uses: pnpm/action-setup@v4
               with:
                 version: 9
             - uses: actions/setup-node@v4
               with:
                 node-version: 20
                 cache: pnpm
             - run: pnpm install --frozen-lockfile
             - name: Format check
               run: pnpm format:check
             - name: Typecheck
               run: pnpm typecheck
             - name: Unit & Integration tests
               run: pnpm --filter @workspace/web test
       ```

    3. Verify root package.json has the required scripts: format:check, typecheck, test:ci. The format:check and typecheck scripts already exist. Confirm `pnpm --filter @workspace/web test` works (depends on Plan 01 adding the test script to web/package.json).

    NOTE: E2E tests are excluded from CI intentionally — they require a running Next.js server with a seeded database and Supabase connection. E2E tests are for local development verification. If CI E2E is desired later, it can be added as a separate workflow with service containers.
