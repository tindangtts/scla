Ticket endpoints (under /api/tickets):
- GET /api/tickets — Bearer token → 200 [ { id, ticketNumber, title, category, serviceType, status, unitNumber, description, attachmentUrl, createdAt, updatedAt, updates } ]
- POST /api/tickets — Bearer token + { title, category, serviceType, description, unitNumber?, attachmentUrl? } → 201 { ticket with ticketNumber: "SA-XXXX" }
- GET /api/tickets/:id — Bearer token → 200 { ticket } or 404
- GET /api/tickets/:id/messages — Bearer token → 200 [ messages ]
- POST /api/tickets/:id/messages — Bearer token + { content } → 201 { message }

Booking endpoints (under /api/bookings):
- GET /api/bookings — Bearer token → 200 [ { id, bookingNumber, facilityId, facilityName, date, startTime, endTime, totalAmount (number), status, paymentStatus, notes, recurringGroupId, createdAt } ]
- POST /api/bookings — Bearer token + { facilityId, date, slotId, notes?, recurring? } → 201 { booking with bookingNumber: "BK-XXXX" }
- GET /api/bookings/:id — Bearer token → 200 { booking } or 404
- POST /api/bookings/:id/cancel — Bearer token → 200 { booking with status: "cancelled" }

Facility endpoints (under /api/facilities, no auth required):
- GET /api/facilities — 200 [ facilities ]
- GET /api/facilities/:id/slots?date=YYYY-MM-DD — 200 [ { id, startTime, endTime, isAvailable, price } ]

Seed ticket data: SA-0001 (in_progress, residentUser), SA-0002 (open, residentUser), SA-0003 (completed, guestUser)
Seed booking data: BK-0001 (upcoming, residentUser), BK-0002 (completed, residentUser), BK-0003 (cancelled, residentUser)
