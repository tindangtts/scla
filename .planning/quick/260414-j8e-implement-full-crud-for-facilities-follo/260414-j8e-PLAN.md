<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin auth | All actions require requireAdmin() — untrusted users cannot reach CRUD |
| FormData input | User-supplied form values cross into DB operations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-fac-01 | S (Spoofing) | actions.ts | mitigate | requireAdmin() on every server action ensures only authenticated admin staff can create/update/delete |
| T-fac-02 | T (Tampering) | actions.ts formData | accept | FormData values are cast to expected types; Drizzle parameterizes all queries preventing SQL injection. Category is cast but only used in a typed pgEnum column (DB rejects invalid values). Low risk for an admin-only surface. |
| T-fac-03 | I (Info Disclosure) | getFacilityById | accept | Facility data is not sensitive (public rates/hours). Admin-only access via requireAdmin(). |
| T-fac-04 | D (Denial of Service) | deleteFacility | accept | Admin-only action; accidental deletion is a training/UX concern, not a security threat. Audit log provides traceability. |
</threat_model>
